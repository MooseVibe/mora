# Design System · Mora

> Это **визуальный закон** проекта. Если ты собираешься написать инлайн цвет, размер, отступ — сначала проверь, есть ли он здесь. Если нет — добавь сюда, потом используй.

## Визуальный вайб

**Мистика, тёмный фон, золотые акценты.** Классика таро-эстетики. Не нежно-пастельное, не футуристично-минималистичное. Это таинство, ритуал, что-то древнее и недосказанное.

Референсы для ощущения:
- Старые колоды Райдер-Уэйт
- Тёмное золото на бордовом / тёмно-синем
- Символика: звёзды (✦), ромбы (◆), луна (☽), римские цифры (XVIII)

## Типографика

Шрифты лежат локально в `public/assets/fonts/` и подключаются через `public/assets/fonts.css`.

| Шрифт | Начертания | Роль |
|---|---|---|
| **Spectral SC** (display serif) | 300, 400, 500 | Заголовки, названия карт, ритуальные элементы |
| **Roboto Condensed** (sans-serif) | 300, 400, 500, 600 | Весь UI: body, кнопки, nav, подписи |

**Правило против скачка шрифтов:** локальные Mora-шрифты, которые видны в первом кадре, dashboard/journal reader или result-screen, не должны использовать `font-display: swap`. При добавлении нового семейства или веса сначала проверь, где он появляется: критичные веса preload-ятся в `src/app/layout.tsx`, а `@font-face` в `public/assets/fonts.css` должен исключать позднюю подмену метрик. Иначе пользователь видит “не тот шрифт”, затем текст и layout скачут после загрузки файла.

- `xs` = 4px

## Тон голоса

Mora говорит **как мудрый собеседник, не как продукт**. Без маркетингового энтузиазма, без восклицательных знаков пачками, без эмодзи.

- ✅ «Карта дня готова. Открой её, когда будешь готов услышать.»
- ❌ «🔥 Скорее тяни карту! Будет интересно!»

- ✅ «Войти»
- ❌ «Войти прямо сейчас!»

- ✅ «Эта карта уже выпала сегодня. Возвращайся завтра.»
- ❌ «Упс! Вы уже использовали попытку.»

Текст должен звучать слегка отстранённо, с лёгкой долей мистики. Никакого корпоративного дружелюбия.

## Состояния

Каждый интерактивный элемент должен иметь:
- **default** — обычное состояние
- **hover** — на десктопе при наведении
- **active / pressed** — в момент клика
- **focus** — для клавиатуры (обязательно, не убирай outline без замены)
- **disabled** — приглушённое, без интерактивности
- **loading** — для асинхронных действий

Не делай экранов без состояний загрузки и пустых состояний. Это плохой UX.

## Анимации — ключевая часть продукта

Это **не оформление**, это часть ценности Mora. У живого таролога есть пауза, движение рук, переворот карты — это создаёт ощущение ритуала. Веб-сервис без этого превращается в «нажал кнопку, обновился экран». Мы так не делаем.

**Правило:** каждое значимое взаимодействие должно ощущаться как часть таро-ритуала, а не как UI-микро-переход. Если ты делаешь экран и видишь только мгновенные смены состояний — ты делаешь его неправильно.

### Иерархия анимаций

**1. Ритуальные** — главные моменты, ради которых пользователь пришёл. Они должны быть длинными, насыщенными, кинематографичными.
- Вытягивание карты: 2-4 секунды весь сценарий (приближение колоды → выбор → переворот → раскрытие)
- Открытие карты в дневнике: 600-1000ms с плавным переворотом
- Появление сводки по 3 картам: каскадное появление одна за другой
- Переход на дашборд после логина: атмосферный fade с задержкой контента

**2. Переходы между экранами** — не мгновенные. Минимум плавный fade или slide, в идеале — связанные через shared elements (карта продолжает движение между экранами).
- Длительность: 400-600ms
- Easing: `cubic-bezier(0.4, 0, 0.2, 1)` или более медитативный

**3. Микро-интеракции** — ховеры, фокусы, нажатия. Короткие, но **никогда не дёрганые**.
- Длительность: 200-300ms
- Лёгкий glow на ховере у интерактивных элементов, плавное масштабирование (1.0 → 1.02)
- Никаких резких скачков цвета — только плавное проявление

### Тайминги (базовые токены)

| Токен | Значение | Где применять |
|---|---|---|
| `--duration-instant` | 150ms | смена иконок, фокус |
| `--duration-quick` | 250ms | ховеры, активные состояния |
| `--duration-base` | 400ms | переходы внутри экрана |
| `--duration-slow` | 700ms | переходы между экранами |
| `--duration-ritual` | 1500-3000ms | вытягивание карты, переворот |

### Easing

| Токен / описание | Кривая | Где |
|---|---|---|
| `--ease` _(в коде)_ | `cubic-bezier(0.22, 0.61, 0.36, 1)` | стандартный ease всего проекта — fade-unit, nav, deck |
| плавный выход | `cubic-bezier(0.16, 1, 0.3, 1)` | открытие панелей, deck hover |
| кнопка нажата | `cubic-bezier(0.22, 0.61, 0.36, 1)` | `transform: scale(0.98)` — active |

**Никаких:** `ease-in` без `out`, bounce, elastic, spring с overshoot. Это не TikTok и не геймдев — это таро.

### Что обязательно анимировано

- **Вытягивание карты** — главный ритуал продукта. Должен ощущаться как событие, а не клик. См. отдельный раздел ниже.
- **Переворот карты** — везде где карта раскрывается (в дашборде, в дневнике)
- **Появление текста описания карты** — постепенное, не мгновенное (fade + лёгкий slide-up, можно с разбивкой на строки)
- **Переходы между экранами** — fade или shared element, не cut
- **Мистические элементы фона** — лёгкое медленное движение, мерцание звёзд, плавающие символы (но не отвлекают)
- **Hover на интерактивных элементах** — золотой glow, лёгкое поднятие, изменение тени
- **Loading-состояния** — не спиннер. Дыхание, пульсация, плавающие символы. Loader тоже должен быть в эстетике.

### Сценарий вытягивания карты (эталон)

Это якорный ритуал продукта. Если делаешь — делай так:

1. **Подготовка** (300-500ms) — приглушение фона, появление колоды или мистического элемента
2. **Выбор** (800-1500ms) — карта медленно поднимается / появляется / выплывает; пользователь видит её рубашку
3. **Переворот** (600-1000ms) — карта переворачивается 3D-эффектом, не plain flip
4. **Раскрытие** (400-600ms) — название карты появляется с задержкой ~200ms, потом описание построчно

Общая длительность ритуала: **2-4 секунды**. Это не «слишком долго» — это правильно. Если пользователь хочет быстро — он не наша аудитория.

### Технические ориентиры

- **Library:** Framer Motion (рекомендуется для React/Next.js) или CSS-анимации для простых случаев
- **3D-эффекты** (переворот карты): `transform: rotateY()` + `transform-style: preserve-3d` + `backface-visibility: hidden`
- **Производительность:** анимируй только `transform` и `opacity`. Не анимируй `width`, `height`, `top`, `left` — это убивает FPS.
- **Reduced motion:** уважай `prefers-reduced-motion`. Для пользователей с этой настройкой — упрощённые версии (fade без движения, без длинных задержек), но **ритуал не убирай полностью** — просто делай его сдержаннее.

### Чего избегать

- Дёрганые переходы без easing
- Bounce и elastic — это не наш характер
- Слишком быстрые анимации (<150ms) на значимых элементах — пользователь не успевает заметить
- Слишком много одновременных анимаций — это утомляет
- Spinner-loader — заменяй на тематический

## Mobile / Desktop

Оба формата равноценны. Минимум:
- Mobile проверять на 375px
- Desktop проверять на 1440px
- Не должно ломаться между этими ширинами

На mobile — крупные тач-таргеты (минимум 44×44px), upper меню сворачивается.
На desktop — используем пространство, не растягиваем мобильный макет на всю ширину.

## Иконография

Используй мистические символы юникода (✦ ◆ ☽ ✕ ◯ XVIII) или тонкие линейные иконки. **Не используй цветные иллюстративные иконки** и эмодзи.
