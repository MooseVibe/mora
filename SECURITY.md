# Security Policy

MORA is an early-stage project, but security reports are welcome and taken seriously.

## Reporting a Vulnerability

Please do not open a public issue for vulnerabilities involving authentication, user data, Supabase configuration, or deployment secrets.

Instead, contact the maintainer through the GitHub profile for this repository and include:

- A short description of the issue
- Steps to reproduce, if possible
- The affected route, component, or file
- Any suggested fix or mitigation

## Scope

Security-sensitive areas include:

- Authentication and session handling
- Supabase client/server configuration
- Card journal data access
- API route authorization
- Environment variables and deployment configuration

## Supported Version

Only the current `main` branch is actively maintained.
